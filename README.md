# FX-tracker: трекер USD/EUR/UAH

Автоматичний збирач курсів валют з трьох банків (Monobank, Райффайзен,
Sense Bank) з аналізом і рекомендаціями щодо моменту конвертації.
Розроблено для сценарію: ФОП у Монобанку, зарплата в USD, накопичення
ділиться між USD і EUR.

## Джерела даних

| Банк       | Як отримуємо                                           | Оновлення      |
|------------|--------------------------------------------------------|----------------|
| Monobank   | Публічний API `api.monobank.ua/bank/currency`          | раз на 5 хв    |
| Райффайзен | Парсинг сторінки minfin (у банку немає публічного API) | раз на годину  |
| Sense Bank | Парсинг сторінки minfin (URL досі під `/alfa-bank/`)   | раз на годину  |

**Чому не скрейпимо сайти банків напряму?** Сайти банків частіше міняють
розмітку і часто захищені від автоматичних запитів. Minfin — стабільний
агрегатор, який десятки сервісів використовують як джерело. Якщо minfin
колись змінить розмітку — побачиш чітку помилку у логах, а не тихий
збій з нулями.

## Що робить

1. **Збирає курси** з трьох банків 3 рази на день.
2. **Зберігає історію** в SQLite (`fx.db`) — один файл, жодних сторонніх сервісів.
3. **Аналізує** поточний курс проти середніх за 7 і 30 днів, рахує
   перцентиль (наскільки сьогодні дорого/дешево в межах місяця).
4. **Рекомендує дію**:
   - 🟢 Конвертувати валюту в грн — якщо банк зараз купує валюту дорого (топ-25%).
   - 🟢 Купити валюту за грн — якщо банк зараз продає валюту дешево (bottom-25%).
   - 🟡 Чекати — якщо курс близький до середнього.
5. **Порівнює три банки** і каже, в якому з них зараз найвигідніше
   купити або продати валюту.

## Що він НЕ робить

- Не прогнозує майбутній курс (цього ніхто не вміє надійно).
- Не виконує операції сам — тільки підказує.
- Не читає твій приватний баланс — лише публічні курси.

## Встановлення

```bash
# 1. Розпакуй у зручну папку, наприклад ~/fx-tracker
cd ~/fx-tracker

# 2. Створи virtualenv (опційно, але рекомендовано)
python3 -m venv venv
source venv/bin/activate          # macOS / Linux
# або:  venv\Scripts\activate     # Windows

# 3. Встанови залежності
pip install -r requirements.txt

# 4. Ініціалізуй БД + перший запуск
python main.py both
```

Перший запуск має показати курси всіх трьох банків. Якщо щось з джерел
недоступне — воно залогується і пропуститься, інші працюватимуть.

## Щоденне використання

```bash
python main.py collect   # тільки зібрати дані (тихо, для cron 3× на день)
python main.py report    # тільки звіт (для cron о 10:00)
python main.py both      # і те, і те (ручний запуск)
```

Після запуску:
- `fx.db` — історія курсів (SQLite).
- `last-report.txt` — останній звіт у зрозумілому вигляді.
- `fx-tracker.log` — журнал роботи.

## Автоматизація

### macOS / Linux (cron)

```bash
crontab -e
```

Додай (замість `/Users/ти/fx-tracker` підстав свій абсолютний шлях):

```cron
# Збір курсів 3 рази на день у робочий час (пн-пт):
# 9:00 — перед початком торгів, 14:00 — середина дня, 19:00 — після закриття
0 9,14,19 * * 1-5  cd /Users/ти/fx-tracker && /Users/ти/fx-tracker/venv/bin/python main.py collect

# Звіт раз на день о 10:00 (по буднях)
0 10 * * 1-5       cd /Users/ти/fx-tracker && /Users/ти/fx-tracker/venv/bin/python main.py report
```

Перевірка, що cron спрацював:
```bash
tail -f ~/fx-tracker/fx-tracker.log     # живий лог
cat    ~/fx-tracker/last-report.txt     # останній звіт
```

Якщо хочеш збирати й у вихідні — заміни `1-5` на `*`.

### Windows (Task Scheduler)

1. Відкрий `Task Scheduler` → `Create Task` (не Basic, а саме Create Task).
2. Вкладка **General**: назви задачу, постав "Run whether user is logged on or not".
3. Вкладка **Triggers**: додай 3 тригери (`New…`):
   - Daily, `09:00:00`, Recur every 1 day
   - Daily, `14:00:00`, Recur every 1 day
   - Daily, `19:00:00`, Recur every 1 day
4. Вкладка **Actions**: `New…`
   - Action: `Start a program`
   - Program: `C:\шлях\до\fx-tracker\venv\Scripts\python.exe`
   - Arguments: `main.py collect`
   - Start in: `C:\шлях\до\fx-tracker`
5. Збережи.
6. Створи ще одну задачу з тригером `Daily 10:00` і аргументом `main.py report`.

### Альтернатива для Linux: systemd timer

`~/.config/systemd/user/fx-collect.service`:
```ini
[Unit]
Description=FX tracker collect

[Service]
Type=oneshot
WorkingDirectory=%h/fx-tracker
ExecStart=%h/fx-tracker/venv/bin/python main.py collect
```

`~/.config/systemd/user/fx-collect.timer`:
```ini
[Unit]
Description=Collect FX rates 3 times a day

[Timer]
OnCalendar=Mon..Fri 09,14,19:00:00
Persistent=true

[Install]
WantedBy=timers.target
```

Аналогічно `fx-report.service` і `fx-report.timer` з `OnCalendar=Mon..Fri 10:00`.

Активація:
```bash
systemctl --user enable --now fx-collect.timer fx-report.timer
systemctl --user list-timers   # перевірка
```

## Як читати звіт

Приклад:

```
Поточні курси:
  Джерело      Пара         Buy (ти продаєш)   Sell (ти купуєш)
  --------------------------------------------------------------------------
  monobank     USD/UAH               43.2000            43.6500
  raiffeisen   USD/UAH               43.3000            43.7200
  sense        USD/UAH               43.3500            43.7000

▸ USD/UAH
  Середнє за 7 днів : buy=42.9106  sell=43.4044
  Середнє за 30 днів: buy=42.9991  sell=43.4975
  Позиція в історії: buy  85.0%  sell  80.0%  (100% = максимум за 30 днів)
  Рекомендація:      🟢 КОНВЕРТУВАТИ ВАЛЮТУ В ГРН
  Банк купує валюту за 43.2000 — це в топ-15% за 30 днів.

Де найвигідніше здійснити операцію ЗАРАЗ:
▸ USD/UAH
  Продати валюту дорожче за все: sense @ 43.3500
  Купити валюту дешевше за все:  monobank @ 43.6500
```

Як читати:
- **"Buy" у банку = ти продаєш** йому валюту. Вищий buy = вигідніше продавати.
- **"Sell" у банку = ти купуєш** у нього валюту. Нижчий sell = вигідніше купувати.
- **Перцентиль 85%** означає: сьогоднішній buy вищий, ніж у 85% днів за
  останній місяць. Тобто зараз сильніший, ніж зазвичай — відносно
  непоганий момент продавати.
- **Порівняння банків** знизу — це конкретна відповідь "куди йти саме зараз".

## Перші два тижні

Перші ~5 днів звіт для пар Моно казатиме **"insufficient_data"** — це
нормально, потрібно накопичити історію (мінімум 5 точок за 30 днів).
Через 2 тижні сигнали стануть репрезентативними, через місяць — повноцінними.

## Як розширити

- **Додати інший банк**: допиши функцію `fetch_xxx()` у `fetchers.py` і
  зареєструй її у `fetch_all` та у `SOURCES` у `main.py`. Для банку, який
  теж є на minfin, достатньо одного рядка в словнику `_MINFIN_URLS`.
- **Telegram-сповіщення**: у кінці `cmd_report()` додай:
  ```python
  import requests
  requests.post(
      f"https://api.telegram.org/bot{TOKEN}/sendMessage",
      json={"chat_id": CHAT_ID, "text": report[:4000]},
  )
  ```
- **Email-звіт**: `smtplib` + gmail SMTP з app password.
- **Графіки**: дані у `fx.db`, відкривається у DB Browser for SQLite або
  через `pandas.read_sql()` → `matplotlib`.

## Стратегія (нагадування)

Коли отримуєш USD на ФОП і хочеш ділити між USD і EUR:

1. **USD залишається на USD-рахунку** — нульова втрата на спредах.
2. **UAH для витрат** — продавай частину доларів, коли курс buy у топ-25%
   за місяць. Приблизно 2-3 рази на місяць, трьома-чотирма траншами.
3. **EUR для накопичень** — якщо Моно / Райф / Sense підтримує прямий
   обмін USD→EUR, це дешевше ніж USD→UAH→EUR (один спред замість двох).
   Звіт показує крос-курс Моно EUR/USD.
4. **DCA замість таймінгу** — розбивай велику операцію на кілька менших.
   Ніхто не знає точку дна/піку, а середнє виходить кращим за спробу
   вгадати "ідеальний день".
5. **Порівнюй банки перед великою операцією** — різниця 0.3% на $5000
   це $15. За рік кілька сотень.
6. **Сезонність зарплати ≠ сезонність курсу.** Не обов'язково міняти
   в день зарплати. Якщо курс на зарплатний день у "червоній зоні", краще
   почекати 1-3 дні.
